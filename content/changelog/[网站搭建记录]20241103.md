# What I Learned

1. **Why Zola uses so many files**

   The `templates/base.html` file uses [Tera](https://github.com/Keats/tera), which is a template engine that defines the basic layout of your site.  
   At the beginning, it imports a bunch of smaller HTML files that act like reusable components.

   This kind of **modular structure** gives you flexibility. For example, inside `base.html`, youâ€™ll often see things like:

   ```html
   {%- block seo %} ... {%- endblock seo %}
   ```

   This tells Zola:  
   *"Hey, this section can be replaced or extended later."*

   Think of each `block` as a **placeholder**. If you donâ€™t override it, Zola uses the default. But if you want to customize it, just create another template that extends the base, usually like this:

   ```html
   {% extends "base.html" %}
   ```
