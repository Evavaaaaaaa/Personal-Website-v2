学到了什么

  1. zola 为什么会用那么多文件： templates/base.html 用 Tera 讲网站的主页应该长什么样。最开始先 import 一大堆其它写好 template functions 的 html >>> 这是一种模块化，灵活度高。比如 base.html 里面有许多 '{%- block seo %}...{%- endblock seo %}' 告诉 zola 这两个 code 之间的内容是可以被替换掉的。换句话说，这个'block'类似一种占位（placeholder）。如果没有重写里面的定义，那么zola 会用原先默认的内容。如果要改变，那只需要另外建立一个 html，扩展这个基础模块（通常使用 '{% extends "base.html" %}' 或类似的命令）。
